🚀 빠른 시작

Flexible Classroom은 Agora RTC SDK, RTM SDK, 인터랙티브 화이트보드 및 클라우드 녹화를 결합한 로우코드 솔루션입니다. 복잡한 오디오/비디오 로직을 깊이 학습하지 않고도 온라인 인터랙티브 교실을 빠르게 통합할 수 있습니다.

💡 주요 장점

  • 개발 시간 대폭 절약
  • 낮은 개발 진입 장벽
  • 엔터프라이즈급 안정성
  • 완전한 커스터마이징 가능

⚙️ 서비스 활성화

📋 개요

Flexible Classroom을 사용하기 전에 Agora 콘솔에서 서비스를 활성화해야 합니다.

활성화 단계

  1. Agora 콘솔 등록 및 로그인
    Agora 공식 웹사이트에서 계정을 생성하고 콘솔에 로그인합니다.
  2. 새 프로젝트 생성
    콘솔에서 새로운 프로젝트를 생성하고 프로젝트 이름을 설정합니다.
  3. App ID 및 임시 토큰 획득
    프로젝트 설정에서 App ID를 확인하고 테스트용 임시 토큰을 생성합니다.
  4. Flexible Classroom 서비스 활성화
    프로젝트 설정에서 Flexible Classroom 서비스를 활성화합니다.

🔧 Flexible Classroom 통합

통합 프로세스

  1. npm을 통한 SDK 설치
    npm 또는 yarn을 사용하여 agora-classroom-sdk 패키지를 설치합니다.
  2. SDK 가져오기
    프로젝트에서 AgoraEduSDK를 import합니다.
  3. SDK 초기화
    App ID와 지역(region) 정보로 SDK를 구성합니다.
  4. 교실 시작
    필요한 매개변수와 함께 launch 메서드를 호출하여 교실을 시작합니다.
  5. HTML 컨테이너 추가
    교실이 렌더링될 DOM 요소를 준비합니다.
참고: 교실 유형은 roomType 매개변수로 설정할 수 있습니다

💡 모범 사례

교실 녹화 모범 사례

녹화 안정성 보장

  1. 수업 전 녹화 기능 테스트
    실제 수업 전에 녹화 기능이 정상적으로 작동하는지 테스트합니다.
  2. 녹화 상태 모니터링
    녹화 중 상태를 실시간으로 모니터링하여 문제를 조기에 발견합니다.
  3. 녹화 실패 재시도 메커니즘 구현
    네트워크 문제 등으로 녹화가 실패할 경우 자동으로 재시도합니다.
  4. 녹화 상태 알림 설정
    녹화 시작, 중지, 오류 등의 상태를 사용자에게 알립니다.
  5. 네트워크 중단 처리
    네트워크가 일시적으로 중단되어도 녹화를 계속할 수 있도록 합니다.

녹화 품질 향상

  1. 적절한 녹화 레이아웃 사용
    수업 유형에 맞는 녹화 레이아웃을 선택합니다.
  2. 합리적인 비디오 비트레이트 및 해상도 설정
    네트워크 상황을 고려하여 최적의 설정을 선택합니다.
  3. 안정적인 네트워크 연결 보장
    유선 연결 사용을 권장하며, 대역폭을 충분히 확보합니다.
  4. 오디오/비디오 매개변수 최적화
    환경에 맞는 오디오 샘플링 레이트와 비디오 프레임 레이트를 설정합니다.
  5. 적절한 녹화 모드 구성
    개별 녹화 또는 합성 녹화 중 필요에 맞는 모드를 선택합니다.

온라인 감독

구현 솔루션

  • 학생 카메라 및 마이크 활성화: 시험 중 학생의 행동을 모니터링합니다.
  • 화면 공유 모니터링 구현: 학생의 화면을 실시간으로 확인합니다.

  • 비정상 행동 감지 구현: AI를 활용하여 부정행위를 자동으로 감지합니다.
  • 실시간 경고 시스템: 의심스러운 행동 발견 시 즉시 감독관에게 알립니다.

기술 포인트: 다중 스트림 구독, 실시간 스냅샷 업로드, AI 기반 행동 분석, 브라우저 탭 전환 감지 등의 기술을 활용합니다.

📚 클라이언트 API

AgoraEduSDK

초기화 및 시작 메서드를 제공하는 메인 SDK 클래스입니다.

config()

설명: SDK 매개변수를 구성합니다.

매개변수:

  • appId (string): Agora App ID
  • region (string, optional): 지역 코드 ('CN', 'NA', 'EU', 'AP')

launch()

설명: 교실을 시작합니다.

매개변수:

  • dom (HTMLElement): 교실이 렌더링될 DOM 요소
  • launchOption (LaunchOption): 교실 시작 옵션

version()

설명: SDK 버전을 가져옵니다.

반환값: SDK 버전 문자열

LaunchOption 주요 매개변수:
  • rtmToken: 서버 인증 토큰
  • userUuid: 사용자 고유 ID
  • userName: 사용자 이름
  • roomUuid: 방 고유 ID
  • roomName: 방 이름
  • roleType: 역할 유형
  • roomType: 교실 유형
  • duration: 수업 시간 (초)
  • language: 언어 설정

❓ FAQ

통합 및 사용 문제

Q: 초기화가 실패하는 이유는 무엇입니까?
A: 다음 사항을 확인하세요:
  • App ID와 토큰이 올바른지 확인
  • 네트워크 연결이 정상인지 확인
  • 브라우저 호환성 확인
  • 콘솔에서 오류 메시지 확인
Q: 교실에 참여할 수 없는 이유는 무엇입니까?
A: 다음 사항을 확인하세요:
  • 방 이름과 사용자 ID가 유효한지 확인
  • 권한이 올바르게 구성되었는지 확인
  • 브라우저가 카메라/마이크 권한을 부여했는지 확인
  • 토큰이 만료되지 않았는지 확인

오디오 및 비디오 관련 문제

Q: 비디오가 없는 이유는 무엇입니까?
A: 다음 사항을 확인하세요:
  • 브라우저에서 카메라 권한 확인
  • 기기가 제대로 작동하는지 확인
  • 비디오가 음소거되지 않았는지 확인
  • 카메라가 다른 앱에서 사용되고 있지 않은지 확인
  • HTTPS 연결 확인 (HTTP에서는 미디어 접근 불가)
Q: 오디오가 없는 이유는 무엇입니까?
A: 다음 사항을 확인하세요:
  • 마이크 권한 확인
  • 기기가 제대로 작동하는지 확인
  • 오디오가 음소거되지 않았는지 확인
  • 다른 브라우저로 테스트
  • 오디오 출력 기기 확인
Q: 비디오가 끊기거나 멈춥니까?
A: 다음 조치를 취하세요:
  • 네트워크 연결 상태 확인
  • 비디오 해상도를 낮춤
  • 불필요한 브라우저 탭 닫기
  • 브라우저 확장 프로그램 비활성화
  • CPU 사용량 확인 (다른 프로그램 종료)
  • 하드웨어 가속 활성화

브라우저 호환성 문제

Q: 어떤 브라우저가 지원됩니까?
A: 지원되는 브라우저:
  • Chrome 58+ (권장)
  • Firefox 56+
  • Safari 11+
  • Edge 79+

최상의 성능을 위해 최신 버전의 Chrome을 권장합니다.

Q: Safari에서 문제가 발생합니까?
A: Safari 사용 시 주의사항:
  • Safari 11 이상 버전 사용
  • 카메라/마이크 권한 명시적으로 허용
  • iOS Safari의 제한 사항 확인
  • 최신 버전으로 업데이트
  • 개인정보 보호 설정 확인
Q: 모바일 브라우저가 지원됩니까?
A: 모바일 브라우저는 제한적으로 지원됩니다. 일부 기능이 제한될 수 있으며, 모바일 기기에서 더 나은 경험을 위해 네이티브 앱을 사용하는 것을 권장합니다.

성능 및 최적화

Q: 많은 학생이 있을 때 성능을 어떻게 최적화합니까?
A: 성능 최적화 방법:
  • 대형 강의 모드 사용 (roomType: 4)
  • 비디오 해상도 및 프레임 레이트 조정
  • 필요한 스트림만 구독
  • 하드웨어 가속 활성화
  • CDN 모드 고려 (대규모 수업)
Q: 네트워크가 불안정할 때 어떻게 대응합니까?
A: 네트워크 불안정 대응:
  • 적응형 비트레이트 활성화
  • 자동 재연결 메커니즘 구현
  • 네트워크 상태 모니터링
  • 사용자에게 네트워크 상태 알림
  • 저품질 모드로 자동 전환
